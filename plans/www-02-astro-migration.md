## Цель

Убрать дублирование разметки (шапка/подвал/`<head>` повторяются в 29 файлах),
дать авторинг контента в Markdown/MDX и подготовить почву для интерактива —
**без регрессии скорости и SEO** и с деплоем через GitHub Pages.

---

## Обоснование: почему Astro, а не React SPA

| Мотивация | React SPA (CSR) | Astro |
|---|---|---|
| Убрать дублирование | ✅ компоненты | ✅ компоненты |
| Знаком React | ✅ | ✅ React-компоненты как острова (`@astrojs/react`) |
| Markdown/MDX контент | ⚠️ через доп. либы | ✅ content collections из коробки |
| Планируется интерактив | ✅ | ✅ острова с `client:*`-гидрацией |
| **SEO** (title/hreflang/JSON-LD по страницам) | ❌ пустой `<div id=root>` | ✅ статический HTML |
| **Вес** | ❌ +45 КБ рантайма всегда | ✅ 0 JS по умолчанию |
| GitHub Pages | ⚠️ нужен SPA-хак `404→index` | ✅ реальные HTML на маршрут |

Astro закрывает все четыре мотивации и **не теряет** преимуществ текущего сайта
(скорость, лёгкость, статика, индексируемость). React используется точечно —
островами там, где нужен интерактив; остальной сайт остаётся нулевым по JS.

## Целевая архитектура

```

├── tsconfig.json

│                           #   logo.svg, og-image.*, apple-touch-icon.*, js/
├── src/

│   ├── components/
│   │   ├── Head.astro      # <head>: SEO, CSP, theme-color, og/twitter, JSON-LD
│   │   ├── Header.astro    # шапка + nav + lang-switch  (был ×29)
│   │   ├── Footer.astro    # подвал                      (был ×29)
│   │   └── islands/        # React-компоненты для интерактива (.tsx)
│   ├── layouts/
│   │   ├── BaseLayout.astro
│   │   └── DocLayout.astro # layout с сайдбаром
│   ├── content/
```

Стек: **Astro 5.x** + `@astrojs/react` + `@astrojs/mdx` + `@astrojs/sitemap`.
Деплой: GitHub Actions (`withastro/action` + `actions/deploy-pages`),
Pages Source = «GitHub Actions». Кастомный домен сохраняется через `public/CNAME`.

## Ключевые решения — ✅ зафиксированы (2026-05-22)

1. **Модель репозитория** ✅ — исходники Astro живут в этом же репо,
   Actions собирает и публикует `dist/`. Сейчас корень репо = сам сайт; после
   миграции корень = исходники, сайт = собранный артефакт. Старые HTML из корня
   удаляются в Ф.9. `dist/`, `.astro/`, `node_modules/` — в `.gitignore`.

2. **URL блог-постов** ✅ — **вариант A: сохранить URL точь-в-точь** через
   `build.format: 'preserve'` и точное размещение файлов — ноль риска для SEO и
   внешних ссылок. (`/blog/2026-05-18-hello-nova.html` остаётся с `.html`.)

3. **Языковая схема** ✅ — **сохранить все URL точь-в-точь** ручным размещением
   страниц (i18n-роутинг Astro не обязателен). Текущая непоследовательность
   (`/` = RU, но `/doc/` = EN — см. аудит [www-01](www-01-prod-hardening.md))
   на миграцию **не трогается**; унификация — отдельное решение позже.

## Что НЕ делать

- **Чистый CSR React SPA** — регрессия SEO и веса, см. таблицу выше.
- **Переписывать `style.css`** — он production-grade, переносится как есть.
- **Менять URL без необходимости** — каждая смена адреса = риск для SEO и
  внешних ссылок. Сохранять точь-в-точь.
- **Унифицировать языковую схему внутри миграции** — отдельное решение,
  отдельный заход.
- **Тяжёлый интерактив в ядре** — Ф.7 после стабильного cutover.

## Cutover — чеклист (Ф.8, выполнить вручную — меняет живой сайт)

Миграция выполнена в ветке `www-02-astro`. Перенос изменён относительно плана:
весь Astro-проект лежит в подкаталоге **`site/`** (корень репо чистый), workflow
собирает с `path: ./site`. Эти шаги делают новый сайт живым:

1. **GitHub Pages Source → «GitHub Actions».**
   Settings → Pages → Build and deployment → Source.
2. **Триггер workflow.** В `.github/workflows/deploy.yml` заменить
   `on:\n  workflow_dispatch:` на:
   ```yaml
   on:
     push:
       branches: [main]
     workflow_dispatch:
   ```
3. **Влить ветку:** `git checkout main && git merge www-02-astro && git push`.
   Push в `main` запустит workflow → сборка `site/` → деплой `site/dist/` в Pages.
4. **Проверить** nv-lang.org: главная `/`, `/en/`, `/doc/`, `/ru/doc/`,
   `/doc/std/vec/`, `/blog/2026-05-18-hello-nova.html`, `/install/`, 404.
5. **Откат** при проблеме: `git revert` merge-коммита + push (старый сайт
   восстановится из истории), либо временно вернуть Pages Source на ветку.

После успешного cutover — снять с плана статус «не задеплоен», закрыть Ф.8;
Ф.7 (интерактив островами) делать отдельным заходом.

## Отклонения от плана (зафиксировано при выполнении)

- **`site/`** — Astro вынесен в подкаталог (корень репо без перемешки). Решение
  пользователя по ходу работы; решение №1 уточнено.
- **Страницы — `.astro` + партиалы, не MDX-коллекции.** Контент перенесён
  дословно (`?raw` + `set:html`): doc-страницы содержат код Nova с `{}` и
  backtick, MDX потребовал бы рискованной чистки. Дублирование (chrome) убрано
  компонентами — основная цель достигнута. Перевод контента в Markdown-коллекции
  возможен позже как отдельное улучшение.
- **`DocLayout` не выделен** — обёртка doc-layout осталась в теле страниц
  (это контент, не дублирующийся хром); хватило одного `BaseLayout`.
- **`@astrojs/sitemap` не используется** — ручной `public/sitemap.xml` из www-01
  точнее (учитывает std-подстраницы без RU). Интеграция убрана из конфига.
