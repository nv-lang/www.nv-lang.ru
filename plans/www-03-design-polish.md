## Цель

Поднять исполнение дизайна до уровня официальных сайтов go.dev /
rust-lang.org / typescriptlang.org — **по ремеслу** (типографика, ритм,
глубина, состояния, тёмная тема, доступность). Без смены бренда (фиолетовый
акцент, лого-звезда), без редизайна структуры — только полировка.

## Аудит текущего состояния (с чистого листа, 2026-05-22)

Скриншоты: главная (свет/тьма, десктоп/мобайл), doc, spec, install.

Сильные стороны: чисто, консистентно, рабочая тёмная тема, аккуратные
окна кода, адаптив (после правок этой сессии). База CSS грамотная —
токены, тёмная тема, `prefers-reduced-motion`, print, skip-link.

Слабые стороны против go/rust/ts:

1. **Плоскость.** Весь сайт — светло-серый/белый, секции сливаются, мало
   глубины. У go/rust/ts — выраженный ритм и контраст блоков.
2. **Hero без акцента.** Заголовок просто чёрным текстом сверху-слева; нет
   визуальной «массы», окно кода висит без подложки.
3. **Карточки фич плоские.** Тонкая рамка, нет тени в покое, скромный
   hover (рамка → акцент, резко).
4. **Монохром.** Один фиолетовый акцент, всё остальное серое — выглядит
   обобщённо. Акцент почти не используется (нет тинтов, градиента, свечения).
5. **Типографика мягкая.** Шкала консервативная; hero-заголовок мелковат
   для своей роли.
6. **Нет `:focus-visible`.** Глобального фокус-кольца нет — это пробел
   доступности и полировки (у go/rust/ts фокус-состояния проработаны).
7. **Тёмная тема плоская.** `--bg` и `--bg-alt` почти неразличимы, карточки
   не читаются как объёмные.
8. **Микровзаимодействия скупые.** Нет hover-подъёма карточек/кнопок,
   нет `::selection`.

## Улучшения (всё — `global.css`, без правки партиалов и разметки)

### Ф.1 — Токены и основа
- Шкала теней `--shadow-sm/md/lg` (мягкие, слоёные) вместо одной.
- Токены акцента: мягкий тинт и свечение для hero/градиентов.
- Тёмная тема: увеличить разделение `--bg` ↔ `--bg-alt` ↔ `--bg-code`.
- `::selection` — акцентный тинт.

### Ф.2 — Доступность и фокус
- Глобальное `:focus-visible` — акцентное кольцо на ссылках, кнопках,
  полях, элементах навигации.

### Ф.3 — Типографика
- Hero-заголовок крупнее и плотнее (`letter-spacing`), увереннее шкала.
- Тонкая настройка межстрочных/отступов заголовков.

### Ф.4 — Hero
- Мягкое радиальное свечение акцента под окном кода — глубина.
- Подложка hero — лёгкий градиент вместо плоского цвета.
- Бейдж статуса — точка-индикатор, аккуратнее.

### Ф.5 — Карточки и компоненты
- Карточки фич: мягкая тень в покое, заметный подъём по hover.
- Иконки фич — выразительнее (контейнер, контраст).
- Кнопки: тень primary в покое, hover-подъём, отшлифованные состояния.
- Окно кода: отшлифованная тень/свечение.
- Status-banner и `.note` — аккуратнее.

### Ф.6 — Ритм секций
- Усилить чередование фонов и вертикальный ритм главной.

## Вне рамок (честно)

- Кастомные иллюстрации/маскот (как у rust) — нужен дизайнер.
- Интерактивный playground (как у ts) — отдельный крупный заход (Ф.7
  плана www-02 уже отметил playground как future).
- Веб-шрифт — системный стек оставляем намеренно (скорость, 0 запросов);
  полировка идёт за счёт шкалы и начертаний, а не смены гарнитуры.
