Решения по дизайну (D-блоки)

Каждое значимое решение по дизайну языка зафиксировано как нумерованный D-блок: вопрос, рассмотренные варианты, принятое решение и обоснование. Спецификация так становится читаемой историей языка, а не просто справочником. Всего зафиксировано более 120 D-блоков.

  • D1 — Парадигма: protocol + data, без классов
  • D2 — Эффекты вместо async/throws/unsafe
  • D52 — Объявления типов: newtype, alias, сумма-типы
  • D53 — Унификация: protocol как kind-токен под type
  • D7 — Один язык, три режима компиляции
  • D6 — Память: управляемая по умолчанию, регионы — опционально
  • D14 — Файбер-рантайм — невидимая инфраструктура
